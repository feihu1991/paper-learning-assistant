配置教程

了解如何配置 OpenClaw 以满足您的需求。

配置文件位置

OpenClaw 的配置文件位于用户主目录下：

配置文件路径 
# macOS / Linux
~/.openclaw/openclaw.json

# Windows
%USERPROFILE%\.openclaw\openclaw.json
💡 提示： 首次运行 openclaw onboard 会自动创建默认配置文件。

配置结构

配置文件采用 JSON 格式，主要包含以下部分：

完整配置示例 
{
  "version": "1.0",
  "gateway": {
    "host": "0.0.0.0",
    "port": 3000
  },
  "channels": {
    "whatsapp": {
      "enabled": true
    },
    "telegram": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN"
    },
    "discord": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN"
    }
  },
  "agents": {
    "default": {
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "YOUR_API_KEY"
    }
  }
}

渠道配置

OpenClaw 支持多种消息渠道，可以在配置文件中启用。

WhatsApp

WhatsApp 配置 
{
  "channels": {
    "whatsapp": {
      "enabled": true,
      "allowFrom": ["+1234567890"]
    }
  }
}
📝 WhatsApp 连接： 首次启用后，需要通过 Web QR 码扫描登录。

Telegram

Telegram 配置 
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz"
    }
  }
}
📝 获取 Telegram Bot Token：
  1. 在 Telegram 中搜索 @BotFather
  2. 发送 /newbot 创建新机器人
  3. 复制获得的 Bot Token

Discord

Discord 配置 
{
  "channels": {
    "discord": {
      "enabled": true,
      "botToken": "YOUR_DISCORD_BOT_TOKEN"
    }
  }
}
📝 获取 Discord Bot Token：
  1. 访问 Discord Developer Portal
  2. 创建新应用 → Bot → 复制 Token
  3. 启用 Message Content Intent

iMessage

iMessage 配置 
{
  "channels": {
    "imessage": {
      "enabled": true
    }
  }
}
📝 iMessage 要求： 需要在 macOS 上运行，并授权辅助功能权限。

代理配置

OpenClaw 支持配置多个 AI 代理，默认使用 Pi agent（RPC 模式）。

默认代理配置

代理配置 
{
  "agents": {
    "default": {
      "type": "pi",
      "model": "claude-3-5-sonnet",
      "rpcUrl": "https://pi-api.claude.ai"
    }
  }
}

自定义 API 提供商

OpenAI 配置 
{
  "agents": {
    "default": {
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "sk-..."
    }
  }
}
Anthropic Claude 配置 
{
  "agents": {
    "default": {
      "provider": "anthropic",
      "model": "claude-3-5-sonnet-20241022",
      "apiKey": "sk-ant-..."
    }
  }
}

多代理配置

多代理示例 
{
  "agents": {
    "support": {
      "type": "pi",
      "model": "claude-3-5-sonnet"
    },
    "coding": {
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "sk-..."
    }
  }
}
💡 提示： 可以为不同渠道或用户分配不同的代理，实现隔离的会话管理。

白名单设置

可以通过白名单限制哪些用户可以与代理交互。

WhatsApp 白名单

WhatsApp 白名单 
{
  "channels": {
    "whatsapp": {
      "enabled": true,
      "allowFrom": ["+1234567890", "+0987654321"]
    }
  }
}

Telegram 白名单

Telegram 白名单 
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "...",
      "allowFrom": [123456789, 987654321]
    }
  }
}
⚠️ 安全建议： 生产环境建议启用白名单功能，只允许授权用户访问。

控制台使用

OpenClaw 提供 Web 控制台进行可视化管理。

打开控制台

打开 Dashboard 
openclaw dashboard

控制台功能

  • 会话管理： 查看和搜索活跃会话
  • 消息调试： 实时查看消息流和代理响应
  • 渠道管理： 连接和配置各个消息渠道
  • 代理配置： 设置和切换 AI 模型
  • 日志查看： 监控系统运行状态
📝 默认地址： 控制台默认在 http://localhost:3000 打开。

环境变量

建议使用环境变量管理敏感配置：

环境变量 
# 添加到 ~/.openclaw/env 或系统环境变量
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
TELEGRAM_BOT_TOKEN=...
DISCORD_BOT_TOKEN=...
⚠️ 安全警告： 请勿将包含 API Key 的配置文件提交到版本控制系统！
