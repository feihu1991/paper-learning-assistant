常见问题

解答您关于 OpenClaw 的常见问题。

通用问题

OpenClaw 是一个开源的多渠道 AI 代理网关平台。它可以连接 WhatsApp、Telegram、Discord、iMessage 等多种消息渠道，并提供强大的 AI 代理能力。

是的，OpenClaw 是完全开源且免费的项目，采用 MIT 许可证。您可以免费用于个人和商业项目。

OpenClaw 支持以下消息渠道：

  • WhatsApp
  • Telegram
  • Discord
  • iMessage (macOS)
  • Web 聊天

您可以通过以下方式获取帮助：

  • GitHub Issues：报告 bug 或提出功能请求
  • GitHub Discussions：社区问答和讨论
  • Discord 服务器：实时交流和帮助

安装问题

这通常是因为安装脚本未正确配置系统路径。解决方法：

macOS / Linux 
# 重新加载配置
source ~/.bashrc  # bash
source ~/.zshrc   # zsh

# 验证安装
which openclaw
openclaw --version

或者手动添加路径到 profile：

echo 'export PATH="$HOME/.openclaw/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

OpenClaw 需要 Node.js 22.0 或更高版本。检查当前版本：

node --version

如果版本过低，请访问 nodejs.org 下载最新 LTS 版本。

Windows 安装可能因为权限或执行策略失败。尝试：

  1. 以管理员身份运行 PowerShell
  2. 检查执行策略： 
    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  3. 重新运行安装脚本： 
    iwr -useb https://openclaw.ai/install.ps1 | iex

引导程序可能需要特定权限或依赖。检查：

  1. 网络连接是否正常
  2. 是否有权限创建系统服务
  3. 查看详细错误日志： 
    openclaw onboard --install-daemon -v

配置问题

配置文件位于用户主目录下：

# macOS / Linux
~/.openclaw/openclaw.json

# Windows
%USERPROFILE%\.openclaw\openclaw.json

确保配置文件是有效的 JSON 格式。可以使用在线 JSON 验证工具检查。

常见错误：

  • 多余的逗号
  • 未闭合的引号或括号
  • 不支持的字段名

修改配置后需要重启 Gateway：

# 重启 Gateway
openclaw gateway restart

渠道问题

WhatsApp 需要通过 Web 界面扫描 QR 码登录：

  1. 确保配置文件已启用 WhatsApp： 
    {
  "channels": {
    "whatsapp": {
      "enabled": true
    }
  }
}
  2. 启动 Gateway 并打开控制台
  3. 在控制台中找到 WhatsApp 连接页面，扫描 QR 码
  4. 检查网络是否稳定

排查步骤：

  1. 确认 Bot Token 正确配置
  2. 检查机器人是否已与 @BotFather 完成设置
  3. 需要启动 Privacy Mode 并禁用 group handling（如果适用）
  4. 验证网络可以访问 Telegram API

可能的原因：

  1. 确认已启用 Message Content Intent
  2. 检查机器人权限是否足够
  3. 确认机器人已添加到服务器
  4. 验证 Bot Token 正确

iMessage 需要：

  1. 在 macOS 上运行
  2. 授予辅助功能权限（系统设置 → 隐私与安全性 → 辅助功能）
  3. 确保已登录 iMessage 账户

代理问题

常见原因：

  1. API Key 填写错误或有多余空格
  2. Key 已过期或被撤销
  3. 账户余额不足
  4. 使用了错误的 API 端点

在配置文件中修改代理配置：

{
  "agents": {
    "default": {
      "provider": "openai",
      "model": "gpt-4",
      "apiKey": "sk-..."
    }
  }
}

支持的模型：OpenAI GPT 系列、Anthropic Claude 系列、Pi Agent 等。

Pi Agent 是 OpenClaw 默认使用的代理模式，通过 RPC 连接到 Infeline 的 Pi API。

{
  "agents": {
    "default": {
      "type": "pi",
      "model": "claude-3-5-sonnet",
      "rpcUrl": "https://pi-api.claude.ai"
    }
  }
}

可能原因和解决方案：

  1. 网络连接问题 - 检查网络或使用代理
  2. API 服务端负载高 - 稍后重试
  3. 请求内容过长 - 减少上下文或分批处理
  4. 增加超时配置
