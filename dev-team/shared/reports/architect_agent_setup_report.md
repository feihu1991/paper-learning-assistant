## Architect Agent 核心能力

### 1. 系统架构设计
- **架构模式选择**: 微服务、单体、事件驱动、分层架构
- **组件设计**: 服务划分、接口设计、数据流设计

### 2. 性能优化
- **系统瓶颈分析**: 识别性能瓶颈点

- **容量规划**: 资源预估和扩容计划
- **可扩展性设计**: 水平和垂直扩展策略

### 3. 技术债务管理
- **债务识别**: 识别代码、设计、测试债务
- **优先级评估**: 评估影响和修复成本

### 4. 系统演进
- **演进路线图**: 短期、中期、长期规划

## 示例任务: 电商平台架构

### 项目概况
- **业务领域**: B2C电商平台
- **规模要求**: 100万日活，1000 TPS
- **质量要求**: 99.99%可用性，P95 < 200ms
- **团队规模**: 20人开发团队
- **时间线**: 3个月MVP，6个月完整功能

### 架构设计
- **架构模式**: 微服务 + 事件驱动
- **服务拆分**: 用户服务、商品服务、订单服务、库存服务、通知服务
- **技术栈**: Node.js/Java/Go混合，PostgreSQL/MongoDB/Redis混合
- **基础设施**: Kubernetes集群，AWS/Aliyun云服务
- **监控告警**: Prometheus + Grafana + ELK

### 成本预估
- **开发成本**: 60人月
- **基础设施**: $3000-5000/月
- **维护成本**: $1000-2000/月
- **第一年总计**: $80,000-120,000
