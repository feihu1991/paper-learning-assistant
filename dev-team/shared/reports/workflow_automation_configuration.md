## 🔧 技术实现

### 监控脚本功能
```javascript
// 主要功能
1. 扫描 tasks/ 目录中的 JSON 任务文件
2. 解析任务类型 (be/fe/qa/devops/architect)
3. 移动任务到 processing/ 目录
4. 创建产出文件到 output/{type}/ 目录
5. 记录处理日志到 logs/ 目录
```

### 自动化流程
```
PM 创建任务 → 监控脚本发现 → 分配给对应 agent → 
移动到 processing/ → 创建产出文件 → 记录日志
```

### 任务处理逻辑
1. **任务发现**: 每分钟扫描一次
2. **任务分配**: 基于任务类型字段
3. **状态跟踪**: 通过目录位置跟踪状态
4. **产出管理**: 结构化产出目录

## 📞 使用指南

### 快速开始
```bash
# 1. 进入工作目录
cd /dev-team/shared

# 2. 运行监控脚本
npm run monitor

# 3. 创建任务文件
# 在 tasks/ 目录创建 JSON 格式的任务文件
```

### 任务文件格式
```json
{
  "task_id": "unique_id",
  "type": "be|fe|qa|devops|architect",
  "objective": "任务目标",
  "specs": { /* 具体规格 */ },
  "priority": "high|medium|low",
  "deadline": "ISO 8601 时间",
  "output_location": "产出路径"
}
```

### 监控和日志
- 监控脚本每分钟自动运行
- 日志文件: `logs/task_monitor.log`
- 产出文件: `output/{type}/` 目录
