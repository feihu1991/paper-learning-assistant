## QA Agent 工作流程

### 监控阶段
1. **任务监控**: 监控 `/dev-team/shared/tasks/` 目录中的 `type: "qa"` 任务
2. **代码获取**: 从 BE/FE 产出目录获取待测试代码
3. **依赖检查**: 检查任务依赖是否完成

### 测试阶段
1. **环境准备**: 安装依赖、配置测试环境
2. **单元测试**: 运行 Jest/Mocha 单元测试
3. **集成测试**: 测试模块间接口
4. **代码检查**: ESLint、代码规范检查
5. **性能测试**: 响应时间、负载测试
6. **安全扫描**: 安全漏洞检查

### 报告阶段
1. **报告生成**: 生成 HTML/JSON/JUnit 测试报告
2. **问题跟踪**: 记录发现的问题和 Bug
3. **验收确认**: 基于测试结果进行验收
4. **产出保存**: 保存报告到 `/dev-team/shared/output/qa/`

## 技术特性

### 自动化测试
- **测试框架**: Jest, Mocha, Cypress, Playwright
- **测试类型**: 单元测试、集成测试、端到端测试
- **覆盖率要求**: ≥ 80% 代码覆盖率

### 代码质量
- **检查工具**: ESLint, Prettier, SonarQube
- **质量门禁**: 无严重错误、复杂度控制、重复代码检查
- **安全扫描**: SQL 注入、XSS、认证绕过等安全检查

### 性能测试
- **测试工具**: k6, Artillery, Lighthouse
- **性能指标**: 响应时间、吞吐量、错误率、资源使用
- **负载测试**: 高并发场景测试

### 报告系统
- **报告格式**: HTML (可视化)、JSON (机器可读)、JUnit (CI/CD 集成)
- **报告内容**: 测试结果、覆盖率、性能指标、安全问题
- **趋势分析**: 历史数据对比和改进建议

## 验收标准

### 质量门禁
1. ✅ 测试通过率 ≥ 95%
2. ✅ 代码覆盖率 ≥ 指定阈值
3. ✅ 性能指标符合要求
4. ✅ 无 Critical/High 级别安全漏洞
5. ✅ 代码质量检查通过

### 报告要求
1. ✅ 完整的测试报告
2. ✅ 详细的问题描述
3. ✅ 具体的改进建议
4. ✅ 可重复的测试步骤
