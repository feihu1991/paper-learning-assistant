## 报告生成规则

### 早报生成流程
1. **获取天气信息** - 使用天气API
2. **获取新闻摘要** - 使用新闻API
3. **读取日程** - 从日历读取
4. **读取任务** - 从任务管理系统读取
5. **生成个性化建议** - 基于用户习惯
6. **格式化输出** - 使用模板格式化

### 晚报生成流程
1. **收集今日数据** - 从时间追踪工具
2. **汇总工作成果** - 从任务管理系统
3. **分析明日计划** - 从日程安排
4. **生成感悟总结** - 基于今日活动
5. **提供晚间建议** - 个性化建议
6. **格式化输出** - 使用模板格式化

### 数据来源
- **天气**: wttr.in 或 Open-Meteo API
- **新闻**: 新闻API或RSS订阅
- **日程**: Google Calendar 或 Outlook
- **任务**: Todoist 或 Trello
- **时间追踪**: Toggl 或 RescueTime

### 个性化设置
用户可以在配置文件中设置：
- 关注的城市
- 感兴趣的新闻类别
- 工作重点领域
- 健康目标
- 学习目标

### 交付方式
- **时间**: 早8:00，晚20:00
- **渠道**: Feishu 消息
- **格式**: Markdown 格式
- **附件**: 天气图表、进度图表等
